Java 基本概念

类加载过程：

1， JVM会先去方法区中找有没有相应类的.class存在。如果有，就直接使用；如果没有，则把相关类的.class加载到方法区

2， 在.class加载到方法区时，会分为两部分加载：先加载非静态内容，再加载静态内容

3， 加载非静态内容：把.class中的所有非静态内容加载到方法区下的非静态区域内

4， 加载静态内容：

4.1、把.class中的所有静态内容加载到方法区下的静态区域内

4.2、静态内容加载完成之后，对所有的静态变量进行默认初始化

4.3、所有的静态变量默认初始化完成之后，再进行显式初始化

4.4、当静态区域下的所有静态变量显式初始化完后，执行静态代码块

5，当静态区域下的静态代码块，执行完之后，整个类的加载就完成了。

Java类的加载、初始化和实例化

Java类的加载、初始化和实例化

类的加载是通过类加载器（Classloader）完成的。

类加载

一、类的加载

我们平常说的加载大多不是指的类加载机制，只是类加载机制中的第一步加载。在这个阶段，JVM主要完成三件事：

1、通过一个类的全限定名（包名与类名）来获取定义此类的二进制字节流（Class文件）。而获取的方式，可以通过jar包、war包、网络中获取、JSP文件生成等方式。

2、将这个字节流所代表的静态存储结构转化为方法区的运行时数据结构。这里只是转化了数据结构，并未合并数据。（方法区就是用来存放已被加载的类信息，常量，静态变量，编译后的代码的运行时内存区域）

3、在内存中生成一个代表这个类的java.lang.Class对象，作为方法区这个类的各种数据的访问入口。这个Class对象并没有规定是在Java堆内存中，它比较特殊，虽为对象，但存放在方法区中。

在迭代过程中，不可以添加元素。

UnsupportedOperationException

Arrays.asList(); // 返回的是固定大小的数组。可以把返回的重新作为参数传递个List或Set

尽量使用Iterator，而不是Enumeration。如果新的类库依旧使用的是Enumeration，可以通过Collections.enumeration(new ArrayList()); // 获得新的Enumeration类

简述对象的创建过程

  1. 创建类的对象，或访问类的静态方法，或类的静态变量
  2. 有关静态初始化的动作都会执行。
  3. 首先new的时候，首先将在堆上分配足够的存储空间
  4. 存储空间清零，对象的成员变量会被设置为默认值
  5. 执行自定义的初始化
  6. 执行构造器

如果需要先进先出的数据结构，应该使用LinkedList。

Stack继承自Vector类

如果进行大量的随机访问，使用ArrayList，如果进行插入或删除使用LinkedList。

队列、双向队列或栈的行为，由LinkedList提供支持。

Set不接受重复元素。

接口中的数据成员都是static和final类型，同样也都是public的。

spring bean注入prototype，该bean依旧是单例

Spring的bean，默认是单例的。如果需要使用多个实例，那么要注入的实例的SCOPE要是SCOPE_PROTOTYPE

一种方法是：

ApplicationContext.getBean()；// 每次调用都能获得一个新服务

另一种是：

​ 使用代理模式。默认情况下是不提供代理的。

​ @Component

​ @Scope(

​ value = ConfigurableBeanFactory.SCOPE_PROTOTYPE, proxyMode = ProxyMode.TARGET_CLASS)

proxyMode有两种模式一个中代理类，一种是代理接口。如果被代理的是接口，那么使用TARGET_INTERFACE
